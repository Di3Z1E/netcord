# Contributing to NetCord

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to NetCord. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
Bugs are tracked as [GitHub issues](https://github.com/Di3Z1E/netcord/issues). Explain the problem and include additional details to help maintainers reproduce the problem.

### Suggesting Enhancements
Enhancement suggestions are tracked as [GitHub issues](https://github.com/Di3Z1E/netcord/issues). Provide a clear and concise description of the suggested enhancement.

### Pull Requests
*   Fill in [the pull request template](.github/pull_request_template.md).
*   Follow the Python style guide (PEP 8).
*   Include unit tests for new functionality.
*   Ensure the CI build passes.

## Development Setup
1. Clone the repo: `git clone https://github.com/Di3Z1E/netcord.git`
2. Install dependencies: `pip install -r requirements.txt`
3. Run tests: `python -m unittest discover tests`

## Style Guide
We use standard Python formatting. Please ensure your code is clean and well-commented.
